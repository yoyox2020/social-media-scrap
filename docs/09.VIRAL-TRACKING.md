## 1. Gambaran Umum

Viral Tracking Intelligence adalah sistem otomatis yang:

1. **Mendeteksi** video YouTube dengan ≥1 juta views dari database
2. **Melacak** channel pemilik video tersebut selama 7 hari (5 video/hari)
3. **Menganalisis** pola komentar — akun yang berkomentar >10x pada video-video channel tersebut dianggap mencurigakan (bot/buzzer)
4. **Mengikuti** akun mencurigakan tersebut dengan membuat tracker baru untuk channel mereka

Sistem ini disebut "intelijen" karena **reaktif** dan **cascading** — tidak perlu input manual, ia mengembangkan pengawasannya sendiri berdasarkan pola perilaku yang ditemukan.

### Penjelasan per file

#### `app/domain/viral_tracking/models.py`

Berisi dua ORM model:

**`ViralChannelTracker`** — satu baris = satu channel yang sedang dipantau

| Kolom | Tipe | Keterangan |
|-------|------|------------|
| `id` | UUID (PK) | Primary key |
| `channel_id` | String | YouTube channel ID (UCxxxxxxxx) |
| `channel_name` | String | Nama channel |
| `trigger_post_id` | UUID (FK → posts) | Post yang memicu tracker dibuat |
| `keyword_id` | UUID (FK → keywords) | Keyword asal post tersebut |
| `tracker_type` | String | `viral` atau `flagged_commenter` |
| `started_at` | DateTime TZ | Waktu tracker dibuat |
| `ends_at` | DateTime TZ | `started_at + 7 hari` |
| `status` | String | `active`, `completed`, `paused` |
| `posts_collected` | Integer | Total post berhasil disimpan |
| `last_scraped_date` | Date | Tanggal scrape terakhir (untuk skip check) |
| `scrape_logs` | JSONB | Log per hari: `[{day, date, posts_new, posts_skipped, error?}]` |

**`FlaggedAccount`** — akun yang komentar >10x

| Kolom | Tipe | Keterangan |
|-------|------|------------|
| `id` | UUID (PK) | Primary key |
| `channel_id` | String | Channel ID akun tersebu |
| `channel_name` | String | Nama akun |
| `comment_count` | Integer | Jumlah komentar yang memicu flag |
| `tracker_id` | UUID (FK → viral_channel_trackers CASCADE) | Tracker yang menemukan akun ini |
| `trigger_post_id` | UUID (FK → posts SET NULL) | Post asal |
| `analysis_tracker_id` | UUID (FK → viral_channel_trackers SET NULL) | Tracker baru yang dibuat untuk menganalisis akun ini |
| `flagged_at` | DateTime TZ | Waktu diflag |

#### `app/services/viral_tracking/service.py`

Business logic murni (tidak ada FastAPI/Celery dependency). Berisi 4 fungsi publik:

```python
# Konstanta
VIRAL_VIEW_THRESHOLD = 1_000_000   # views minimum
TRACKER_DAYS         = 7           # durasi tracking (hari)
POSTS_PER_DAY        = 5           # max video per scrape
COMMENTER_FLAG_THRESHOLD = 10      # min komentar untuk flag
```

**`detect_and_create_trackers(db)`**
- Query semua post YouTube dengan views ≥ 1 juta
- Skip post yang sudah ada trackernya (`trigger_post_id` check)
- Skip channel yang sudah punya tracker aktif (deduplication)
- Buat `ViralChannelTracker` baru per channel baru
- Return list UUID tracker yang dibuat

**`run_daily_channel_scrape(db, tracker_id)`**
- Guard: skip jika `last_scraped_date == today` atau tracker expired
- Scrape via `EnsembleDataClient → YouTubeConnector.get_channel_videos(channel_id, depth=1)`
- Simpan max 5 post baru, tandai dengan `metadata['tracker_id']` dan `metadata['source'] = 'viral_tracking'`
- Tulis log ke `scrape_logs` JSONB via `flag_modified()`
- Update `last_scraped_date = today` dan `posts_collected += new_count`

**`check_and_flag_commenters(db, tracker_id)`**
- Query komentar pada semua post dengan `metadata->>'tracker_id' = tracker_id`
- Group by author+channel_id, filter HAVING COUNT > 10
- Buat `FlaggedAccount` + `ViralChannelTracker` baru (type=`flagged_commenter`) per akun mencurigakan

**`resume_active_trackers(db)`**
- Tandai tracker expired (`ends_at < now`) → status = `completed`
- Return list tracker aktif yang belum scraping hari ini

#### `app/workers/viral_tracking_worker.py`

4 Celery tasks yang membungkus service functions:

| Task | Nama Celery | Retry |
|------|-------------|-------|
| `detect_viral_posts_task` | `workers.viral_tracking.detect_viral_posts` | max 2x |
| `viral_tracking_daily_check_task` | `workers.viral_tracking.daily_check` | max 1x |
| `viral_channel_daily_scrape_task` | `workers.viral_tracking.channel_daily_scrape` | max 3x, delay 5 menit |
| `check_flagged_commenters_task` | `workers.viral_tracking.check_flagged_commenters` | max 2x |

Semua task menggunakan pola `_get_fresh_session()` — membuat engine baru agar tidak ada konflik event loop asyncpg di Celery forked workers.

## 4. Arsitektur Sistem

### Alur trigger otomatis

```
┌──────────────────────────────────────────────────────────┐
│  CELERY BEAT SCHEDULES                                   │
│                                                          │
│  1. Setiap 6 jam (00:00, 06:00, 12:00, 18:00)           │
│     → detect_viral_posts_task                           │
│                                                          │
│  2. Setiap hari jam 03:00                                │
│     → viral_tracking_daily_check_task                   │
└───────────────┬──────────────────────────────────────────┘
                │
                ▼
┌──────────────────────────────────────────────────────────┐
│  detect_viral_posts_task                                 │
│                                                          │
│  1. SELECT posts WHERE views >= 1.000.000                │
│     AND channel belum punya tracker aktif                │
│  2. Untuk tiap channel baru:                             │
│     → INSERT viral_channel_trackers (status=active)      │
│     → queue viral_channel_daily_scrape_task(tracker_id)  │
└───────────────┬──────────────────────────────────────────┘
                │
                ▼
┌──────────────────────────────────────────────────────────┐
│  viral_channel_daily_scrape_task(tracker_id)             │
│                                                          │
│  Guard: last_scraped_date == today? → return (skip)      │
│  Guard: ends_at < now? → status=completed, return        │
│                                                          │
│  EnsembleData API:                                       │
│    GET /youtube/channel/videos                           │
│      ?browseId=UCxxx&depth=1&cursor=                     │
│    → parse richItemRenderer → content → videoRenderer    │
│    → ambil max 5 video terbaru                           │
│    → INSERT posts (metadata tracker_id, source)          │
│                                                          │
│  UPDATE viral_channel_trackers:                          │
│    last_scraped_date = today                             │
│    posts_collected += new_count                          │
│    scrape_logs = append({day, date, posts_new, ...})     │
│                                                          │
│  → queue check_flagged_commenters_task(tracker_id)       │
└───────────────┬──────────────────────────────────────────┘
                │
                ▼
┌──────────────────────────────────────────────────────────┐
│  check_flagged_commenters_task(tracker_id)               │
│                                                          │
│  SELECT author, channel_id, COUNT(*)                     │
│  FROM comments JOIN posts                                │
│  WHERE posts.metadata->>'tracker_id' = tracker_id        │
│  GROUP BY author, channel_id                             │
│  HAVING COUNT(*) > 10                                    │
│                                                          │
│  Untuk tiap akun mencurigakan:                           │
│  → INSERT flagged_accounts                               │
│  → Jika channel_id valid (UCxxx):                        │
│    → INSERT viral_channel_trackers (type=flagged_comm.)  │
│    → queue viral_channel_daily_scrape_task (rekursif!)   │
└──────────────────────────────────────────────────────────┘
```

### Alur daily resume (03:00)

```
viral_tracking_daily_check_task
  ↓
  resume_active_trackers(db):
    → UPDATE trackers SET status='completed'
      WHERE status='active' AND ends_at < now
    → SELECT trackers WHERE status='active'
      AND last_scraped_date != today
  ↓
  → queue viral_channel_daily_scrape_task per tracker
```

## 8. Troubleshooting & Bugs yang Ditemukan

### Bug 1: EnsembleData `channel/videos` butuh parameter `depth`

**Gejala:** HTTP 422 Unprocessable Entity  
**Error:** `{"detail":[{"loc":["query","depth"],"msg":"field required"}]}`  
**Fix:** Tambah `depth=1` sebagai parameter default di `get_channel_videos()`

```python
# connector.py — SEBELUM (bug)
params={"browseId": browse_id, "cursor": cursor}

# SESUDAH (fix)
params={"browseId": browse_id, "cursor": cursor, "depth": min(depth, MAX_DEPTH)}
```

---

### Bug 2: `extract_posts` tidak handle format `richItemRenderer`

**Gejala:** Scrape berhasil (HTTP 200) tapi `new_posts: 0` selalu  
**Penyebab:** Channel/videos endpoint mengembalikan format berbeda dari search:
- Search: `data.results[*].videoRenderer`
- Channel/videos: `data.videos[*].richItemRenderer.content.videoRenderer`

**Fix:** Tambah unwrap layer `richItemRenderer` di `extract_posts()`

```python
# connector.py — SESUDAH (fix)
for item in posts:
    vr = item.get("videoRenderer")
    if not vr:
        rich = item.get("richItemRenderer") or {}
        vr = (rich.get("content") or {}).get("videoRenderer")
    if vr:
        extracted.append(vr)
```

---

### Bug 3: `c.metadata_` di raw SQL harus `c.metadata`

**Gejala:** `check_flagged_commenters` task retry terus, error:  
`UndefinedColumnError: column c.metadata_ does not exist`  
**Penyebab:** SQLAlchemy ORM menggunakan attribute `metadata_` (trailing `_` untuk menghindari konflik reserved name), tapi kolom di PostgreSQL bernama `metadata`.  
**Fix:** Semua raw SQL TEXT menggunakan `c.metadata` dan `p.metadata`, bukan `c.metadata_`.

---

### Bug 4: Deduplication — 158 tracker untuk channel yang sama

**Gejala:** `detect_viral_posts_task` membuat satu tracker per post viral, bukan per channel. Dream Perfect Regime dengan 50+ video viral → 50+ tracker.  
**Fix:** Tambah check `tracked_channels: set[str]` sebelum membuat tracker baru:

```python
# Cek channel yang sudah punya tracker aktif
existing_active_channels = await db.execute(
    select(ViralChannelTracker.channel_id).where(
        ViralChannelTracker.status == "active"
    )
)
tracked_channels: set[str] = set(existing_active_channels.scalars().all())

for post in viral_posts:
    channel_id = _extract_channel_id_from_raw(post.raw_data or {})
    if channel_id in tracked_channels:
        continue  # Skip: sudah punya tracker aktif
    tracked_channels.add(channel_id)
    # ... buat tracker baru
```

---

### Bug 5: SQLAlchemy `flag_modified()` diperlukan untuk JSONB mutation

**Gejala:** `last_scraped_date` ter-update tapi `scrape_logs` tetap `[]` setelah error  
**Penyebab:** SQLAlchemy tidak mendeteksi perubahan pada kolom JSONB secara otomatis ketika list di-reassign.  
**Fix:** Selalu panggil `flag_modified(tracker, "scrape_logs")` setelah memodifikasi JSONB column.

```python
from sqlalchemy.orm.attributes import flag_modified

tracker.scrape_logs = new_logs   # assign baru tidak cukup
flag_modified(tracker, "scrape_logs")  # sinyal eksplisit ke SQLAlchemy
await db.commit()
```

---

### Bug 6: FastAPI route ordering — `/videos/viral` vs `/videos/{video_id}`

**Gejala:** `GET /videos/viral` mengembalikan 404 "Video not found: viral"  
**Penyebab:** FastAPI mendaftarkan route secara berurutan. `GET /videos/{video_id}` yang didaftarkan sebelum `/videos/viral` menangkap string "viral" sebagai path parameter.  
**Fix:** Daftarkan semua literal routes (`/videos/viral`, `/videos/popular`, `/videos/date-search`) **sebelum** parameterized route (`/videos/{video_id}`).
