# API Specification

**Social Intelligence Platform — v1.0**

Base URL: `http://localhost:8000`

API prefix: `/api/v1`

---

## Konvensi Response

### Success
```json
{
  "success": true,
  "data": { ... }
}
```

### Error
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Deskripsi error"
  }
}
```

### HTTP Status Codes

| Code | Keterangan |
|------|------------|
| 200 | OK |
| 202 | Accepted (async task diterima) |
| 400 | Bad Request |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not Found |
| 422 | Validation Error |
| 429 | Rate Limit Exceeded |
| 500 | Internal Server Error |

---

## Authentication

Semua endpoint (kecuali `/health`, `/auth/register`, `/auth/login`) membutuhkan header:

```
Authorization: Bearer <jwt_token>
```

Token didapat dari `POST /api/v1/auth/login`.

## Health

### GET /health

Cek status semua infrastruktur.

**Response 200 — semua OK:**
```json
{
  "success": true,
  "data": {
    "status": "ok",
    "version": "1.0.0",
    "checks": {
      "database":      { "status": "ok" },
      "redis":         { "status": "ok" },
      "ollama":        { "status": "ok" },
      "elasticsearch": { "status": "ok" }
    }
  }
}
```

**Response 207 — sebagian degraded:**
```json
{
  "success": true,
  "data": {
    "status": "degraded",
    "checks": {
      "database":  { "status": "ok" },
      "ollama":    { "status": "error", "detail": "connection refused" }
    }
  }
}
```

Header `X-Request-ID` selalu disertakan pada setiap response.

## Auth

### POST /api/v1/auth/register

Daftarkan user baru.

**Request:**
```json
{
  "email": "user@example.com",
  "password": "strongpassword",
  "full_name": "Nama Lengkap"
}
```

**Response 201:**
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "email": "user@example.com",
    "full_name": "Nama Lengkap"
  }
}
```

---

### POST /api/v1/auth/login

Login dan dapatkan JWT token.

**Request:**
```json
{
  "email": "user@example.com",
  "password": "strongpassword"
}
```

**Response 200:**
```json
{
  "success": true,
  "data": {
    "access_token": "eyJ...",
    "token_type": "bearer",
    "expires_in": 1800
  }
}
```

## Collector

### POST /api/v1/collectors/collect

Trigger pengumpulan data untuk satu keyword. Berjalan async via Celery.

**Request:**
```json
{
  "keyword_id": "uuid",
  "platforms": ["tiktok", "youtube"],
  "max_pages": 5
}
```

**Response 202:**
```json
{
  "success": true,
  "data": {
    "keyword_id": "uuid",
    "job_id": "celery-task-id"
  }
}
```

---

### GET /api/v1/collectors/jobs/{job_id}

Cek status Celery job.

**Response 200:**
```json
{
  "success": true,
  "data": {
    "job_id": "celery-task-id",
    "status": "SUCCESS",
    "result": { ... }
  }
}
```

Status: `PENDING` | `STARTED` | `SUCCESS` | `FAILURE` | `RETRY`

  "force_reprocess": false

**Request:**

  "run_sentiment": true,
  "run_ner": true,

## Trends

### GET /api/v1/trends/keyword/{keyword_id}

Volume post per periode waktu. **Cache 5 menit.**

**Query params:**
- `period`: `day` | `week` | `month` (default: `day`)
- `platform`: filter platform (opsional)

**Response 200:**
```json
{
  "success": true,
  "data": {
    "keyword_id": "uuid",
    "period": "day",
    "total_posts": 1250,
    "volume": [
      { "period": "2026-06-20", "count": 310 },
      { "period": "2026-06-21", "count": 415 },
      { "period": "2026-06-22", "count": 525 }
    ]
  }
}
```

---

## Search

### GET /api/v1/search/

Semantic search pada post yang sudah diembedding.

**Query params:**
- `q` (wajib): query teks
- `keyword_id` (opsional): filter keyword
- `limit`: default 10

**Response 200:**
```json
{
  "success": true,
  "data": {
    "query": "dampak kenaikan bbm",
    "results": [
      {
        "post_id": "uuid",
        "content": "...",
        "score": 0.87,
        "platform": "tiktok",
        "published_at": "2026-06-22T10:00:00"
      }
    ]
  }
}
```

## Agents

> Rate limited: **10 request per 60 detik per user.**
> Response header: `Retry-After: <seconds>` jika 429.

### POST /api/v1/agents/ask

Kirim pertanyaan ke multi-agent pipeline. Berjalan **async** via Celery.

**Request:**
```json
{
  "question": "Apa sentimen publik terhadap kebijakan BBM minggu ini?",
  "keyword_id": "uuid",
  "platform": null
}
```

**Response 202:**
```json
{
  "success": true,
  "data": {
    "keyword_id": "uuid",
    "job_id": "celery-task-id"
  }
}
```

Hasil diambil via `GET /api/v1/collectors/jobs/{job_id}`.

---

### POST /api/v1/agents/ask-sync

Sama seperti `/ask` tapi **sinkron** — tunggu response langsung. Cocok untuk dev/debug.

**Request:** sama dengan `/ask`.

**Response 200:**
```json
{
  "success": true,
  "data": {
    "question": "...",
    "answer": "Berdasarkan 1.250 post dari 3 platform...",
    "agents_used": ["planner", "search", "sentiment", "trend", "summary"],
    "keyword_id": "uuid",
    "sources_count": 25
  }
}
```

## Reports

### POST /api/v1/reports/generate

Trigger generate laporan async.

**Request:**
```json
{
  "keyword_id": "uuid",
  "project_id": "uuid",
  "title": "Laporan Sentimen BBM - Juni 2026",
  "format": "pdf",
  "period": "week",
  "include_posts_sample": true,
  "posts_sample_size": 5
}
```

`format`: `json` | `pdf` | `docx`

**Response 202:**
```json
{
  "report_id": "uuid",
  "job_id": "celery-task-id",
  "status": "pending",
  "format": "pdf"
}
```

---

### POST /api/v1/reports/generate-sync

Generate laporan sinkron (tanpa Celery) — untuk testing.

---

### GET /api/v1/reports/

List laporan milik satu project.

**Query params:** `project_id` (wajib), `limit`, `offset`

---

### GET /api/v1/reports/{report_id}

Detail laporan termasuk preview data JSON.

**Response 200:**
```json
{
  "id": "uuid",
  "title": "Laporan Sentimen BBM - Juni 2026",
  "summary": "Sentimen positif mendominasi (60%)...",
  "format": "pdf",
  "status": "done",
  "keyword_id": "uuid",
  "file_path": "/app/reports/report_uuid.pdf",
  "data": { ... },
  "created_at": "2026-06-23T08:00:00",
  "updated_at": "2026-06-23T08:02:15"
}
```

`status`: `pending` | `processing` | `done` | `error`

---

### GET /api/v1/reports/{report_id}/download

Download file laporan.

**Response:** file binary dengan header `Content-Disposition: attachment`.

| Format | Content-Type |
|--------|-------------|
| json | `application/json` |
| pdf | `application/pdf` |
| docx | `application/vnd.openxmlformats-officedocument.wordprocessingml.document` |

---

### DELETE /api/v1/reports/{report_id}

Hapus record laporan dari DB. File di disk tidak dihapus.

**Response 204:** no content.

## Error Codes

| Code | Keterangan |
|------|------------|
| `VALIDATION_ERROR` | Input tidak valid |
| `NOT_FOUND` | Resource tidak ditemukan |
| `UNAUTHORIZED` | Token tidak valid atau expired |
| `FORBIDDEN` | Tidak punya akses ke resource ini |
| `RATE_LIMIT_EXCEEDED` | Terlalu banyak request, lihat `Retry-After` header |
| `INTERNAL_ERROR` | Error tidak terduga di server |
| `JOB_NOT_FOUND` | Celery job ID tidak ditemukan |
